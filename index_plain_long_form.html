Architectural and Application improvements

There were no good architecture diagrams or documentation of the existing system (apart from what was in people's heads). We used the 3Cs format for visualising the architecture of the existing system. This allowed us to reason about the existing system and arrive at an approach of how to break up the system - which bits to tackle and which bits to leave alone. As well as using this for our project these diagrams were then used by the rest of the department and expanded upon to illustrate the entire system
The creation of this new VOD Publishing module meant we built a group of 3rd party adaptors for CISCO, VUBIQUITY, etc. The old code base tried to have a shared schema for the XML that was sent to these 3 parties and was them modified for each of them individually by taking out which elements they weren't interested in. We changed the model from an ‘opt out’ to and ‘opt in’ model. So rather than changing all the adaptors every time something new was added and to have to regression test these, it was flipped to only add it to the adaptor that the change was for. This would greatly reduce the impact of changes going forward and reduce the number of modules that would need regression tests changes for each new field that was added.
